# 점진적 클러스터링 패턴 (우체국 직원 방식)

## 개념
대량 뉴스를 한번에 처리하지 않고, 10개씩 배치로 나눠 기존 클러스터에 배정/신규생성

## 구현 흐름
1. 뉴스 N개를 10개씩 배치로 분할
2. 각 배치마다:
   - LLM에 뉴스 10개 + 기존 클러스터 요약 목록 전달
   - LLM이 각 뉴스를 기존 클러스터에 배정 또는 새 클러스터 생성 결정
   - 결과로 클러스터 목록 업데이트
3. 모든 배치 처리 후 최종 클러스터 목록 반환

## 장점
- 컨텍스트 윈도우 제한 해결 (호출당 ~5K 토큰)
- 8K 컨텍스트 모델로도 충분히 처리 가능
- 점진적 업데이트 가능 (새 뉴스 추가 시 전체 재처리 불필요)

## 주의사항
- 배치 순서에 따라 결과가 달라질 수 있음
- 첫 배치는 무조건 새 클러스터 생성
- 클러스터 요약이 길어지면 컨텍스트 초과 → 요약 압축 필요

## 실적
- 315뉴스 → 32배치 → 34클러스터 (약 20분)
